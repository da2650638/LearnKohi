# 变量引用/插值语法速查表（bat / sh / VSCode / CMake / MSBuild）

> 目标：看到 `${...}` / `%...%` / `$...` / `$(...)` 这种符号，立刻知道是谁、怎么用、在哪一层生效。

---

## 1) Windows Batch (.bat/.cmd)

### 环境变量
```bat
%VULKAN_SDK%
%PATH%
````

### FOR 循环变量（必须用 %%）

```bat
FOR %%f IN (*.c) DO ECHO %%f
```

### 延迟展开（需要 EnableDelayedExpansion）

```bat
!var!
```

### 注意

* `%VAR%`：一般变量展开
* `%%f`：FOR 变量（双 % 只在 bat 文件里；在 cmd 交互里是 `%f`）

---

## 2) POSIX Shell (bash/zsh/sh)

### 变量

```sh
$HOME
$VULKAN_SDK
```

### 推荐安全写法（带大括号）

```sh
${HOME}
${VULKAN_SDK}
```

### 命令替换

```sh
$(command)
# 旧写法：`command`
```

### 特殊变量（常用）

```sh
$0   # 脚本名
$1   # 第1个参数
$@   # 所有参数（推荐）
$?   # 上个命令退出码
```

---

## 3) VSCode（tasks.json / launch.json / settings.json）

> VSCode 的变量是 **字符串替换**（在 VSCode 里解析），不是 shell/CMake 自己的变量。

### 工作区相关

```text
${workspaceFolder}
${workspaceFolderBasename}
```

### 当前文件相关

```text
${file}
${fileBasename}
${fileDirname}
${fileExtname}
```

### 选中内容/光标

```text
${selectedText}
${lineNumber}
```

### 环境变量（VSCode 语法）

```text
${env:VULKAN_SDK}
${env:PATH}
```

### 输入变量（launch/tasks 里常见）

```text
${input:myVar}
```

### 命令变量（扩展提供）

```text
${command:cmake.buildDirectory}
```

---

## 4) CMake

> CMake 的变量引用是 **`${VAR}`**（配置阶段展开），但也有生成器表达式是 `$<...>`（生成阶段/构建系统阶段）。

### 普通变量（最常用）

```cmake
${CMAKE_SOURCE_DIR}
${PROJECT_NAME}
${CMAKE_CXX_COMPILER}
```

### 环境变量（注意是 ENV{...}）

```cmake
$ENV{VULKAN_SDK}
$ENV{PATH}
```

### 生成器表达式（Generator Expression）

```cmake
$<CONFIG>
$<TARGET_FILE:MyTarget>
$<IF:$<CONFIG:Debug>,debug,release>
```

### 注意

* `${VAR}`：CMake 变量
* `$ENV{VAR}`：环境变量
* `$<...>`：生成器表达式（不要和 shell 的 `$()` 搞混）

---

## 5) Visual Studio / MSBuild（.vcxproj / .props）

> MSBuild 变量基本是 `$(...)`，条件里常见 `'%(...)'`（元数据）。

### 属性（Property）

```xml
$(Configuration)
$(Platform)
$(ProjectDir)
$(OutDir)
$(TargetPath)
```

### Item 元数据（Metadata）

```xml
%(Filename)
%(Extension)
%(FullPath)
```

### 环境变量（MSBuild 内置）

```xml
$(VULKAN_SDK)
```

（如果环境变量存在，MSBuild 通常能映射成同名属性；但更稳的是用 `$(VULKAN_SDK)` 来读）

---

## 6) Makefile（顺手给你加上，最常见也最容易混）

### 变量引用

```make
$(CC)
$(CFLAGS)
$(TARGET)
```

### 自动变量（规则里常用）

```make
$@   # 目标
$<   # 第一个依赖
$^   # 所有依赖
```

---

# 7) 一眼识别：看到符号就知道是哪层

| 语法                   | 大概率来自              | 典型场景                |
| -------------------- | ------------------ | ------------------- |
| `%VAR%`              | Windows bat 环境变量   | build.bat           |
| `%%f`                | Windows bat FOR 变量 | FOR 循环              |
| `!VAR!`              | Windows bat 延迟展开   | 循环拼接变量              |
| `$VAR` / `${VAR}`    | shell 变量           | build.sh            |
| `$(cmd)`             | shell 命令替换         | `$(nproc)`          |
| `${workspaceFolder}` | VSCode 变量          | tasks/launch        |
| `${VAR}`             | CMake 普通变量         | CMakeLists          |
| `$ENV{VAR}`          | CMake 环境变量         | Vulkan SDK          |
| `$<...>`             | CMake 生成器表达式       | 多配置/target 路径       |
| `$(Configuration)`   | MSBuild 属性         | vcxproj/props       |
| `%(Filename)`        | MSBuild 元数据        | ItemGroup/ClCompile |
| `$(CC)`              | Makefile 变量        | Makefile            |

---

# 8) 最关键的“别混淆”清单（只记这 6 条）

1. VSCode 的 `${...}` 是 VSCode 先替换，再交给 shell/程序执行
2. CMake 的 `${...}` 是 CMake 配置阶段替换（不是 VSCode）
3. CMake 环境变量是 `$ENV{...}`，不是 `${...}`
4. CMake 生成器表达式是 `$<...>`，不是 shell 的 `$(...)`
5. MSBuild 是 `$(...)`，不是 CMake 的 `${...}`
6. bat 的 FOR 变量在脚本里必须是 `%%`（交互 cmd 才是 `%`）

```
---
比如 Vulkan SDK：
- bat：`%VULKAN_SDK%\Include`
- sh：`$VULKAN_SDK/include`
- VSCode：`${env:VULKAN_SDK}\Include`
- CMake：`$ENV{VULKAN_SDK}/Include`
- MSBuild：`$(VULKAN_SDK)Include`（或 `$(VULKAN_SDK)\Include`）
```
