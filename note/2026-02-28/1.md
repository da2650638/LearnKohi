# C/C++ 编译器 & 平台 宏速查表

---

## ===============================
## Compiler Macros（编译器识别）
## ===============================

| 编译器 | 宏 |
|--------|------|
| MSVC (Visual Studio) | `_MSC_VER` |
| Clang | `__clang__` |
| GCC | `__GNUC__` |
| MinGW 32-bit | `__MINGW32__` |
| MinGW 64-bit | `__MINGW64__` |

### ⚠ 注意

- ❌ 没有 `__gcc__`
- GCC 正确宏是 `__GNUC__`
- Clang 同时定义：
  - `__clang__`
  - `__GNUC__`

### 推荐判断顺序

```c
#if defined(__clang__)
    // Clang
#elif defined(__GNUC__)
    // GCC
#elif defined(_MSC_VER)
    // MSVC
#else
    #error Unknown compiler
#endif
```
# 编译器 + ABI + 字节序 + 架构检测宏全集

## 编译器检测（Compiler）

### 判断顺序（必须记住）

> **Clang 会定义 `__GNUC__`**，所以必须先判 Clang。

```c
#if defined(__clang__)
  #define KCOMPILER_CLANG 1
#elif defined(__GNUC__)
  #define KCOMPILER_GCC 1
#elif defined(_MSC_VER)
  #define KCOMPILER_MSVC 1
#else
  #error Unknown compiler
#endif
```

###  宏速查表

| 编译器 | 主要宏 | 常见辅助宏 |
|---|---|---|
| MSVC | `_MSC_VER` | `_MSC_FULL_VER`, `_MSC_BUILD`, `_MSVC_LANG` |
| Clang | `__clang__` | `__clang_major__`, `__clang_minor__`, `__clang_patchlevel__` |
| GCC | `__GNUC__` | `__GNUC_MINOR__`, `__GNUC_PATCHLEVEL__` |
| MinGW (GCC/Clang on Windows) | `__MINGW32__` / `__MINGW64__` | 同时也可能有 `__GNUC__` 或 `__clang__` |
| Intel (经典 ICC) | `__INTEL_COMPILER` |  |
| Intel (LLVM ICX) | `__INTEL_LLVM_COMPILER` | 通常也定义 `__clang__` |


### C++ 标准版本检测

| 编译器      | 宏             | 说明                                                |
| -------- | ------------- | ------------------------------------------------- |
| 标准 C++   | `__cplusplus` | 常见值：`201103L, 201402L, 201703L, 202002L, 202302L` |
| MSVC 更可靠 | `_MSVC_LANG`  | 在某些设置下比 `__cplusplus` 更可信                         |

```cpp
#if defined(_MSC_VER)
  #define KCPP_VER _MSVC_LANG
#else
  #define KCPP_VER __cplusplus
#endif
```

## 字节序检测（Endianness）

### GCC/Clang（推荐）

| 宏                         | 说明    |
| ------------------------- | ----- |
| `__BYTE_ORDER__`          | 当前字节序 |
| `__ORDER_LITTLE_ENDIAN__` | 小端常量  |
| `__ORDER_BIG_ENDIAN__`    | 大端常量  |

```cpp
#if defined(__BYTE_ORDER__) && defined(__ORDER_LITTLE_ENDIAN__) && defined(__ORDER_BIG_ENDIAN__)
  #if (__BYTE_ORDER__ == __ORDER_LITTLE_ENDIAN__)
    #define KENDIAN_LITTLE 1
  #elif (__BYTE_ORDER__ == __ORDER_BIG_ENDIAN__)
    #define KENDIAN_BIG 1
  #else
    #error Unknown endianness
  #endif
#else
  // 兜底：多数现代桌面/移动平台都是小端（但引擎底座建议这里直接 #error）
  #error Cannot detect endianness
#endif
```

### Windows（几乎总是小端）

```cpp
#if defined(_WIN32)
  #define KENDIAN_LITTLE 1
#endif
```

| 标准库       | 宏                   |
| --------- | ------------------- |
| libc++    | `_LIBCPP_VERSION`   |
| libstdc++ | `__GLIBCXX__`       |
| MSVC STL  | `_MSVC_STL_VERSION` |

| libc           | 宏                                  |
| -------------- | ---------------------------------- |
| glibc          | `__GLIBC__`（常见）                    |
| musl           | `__musl__`（有些环境可用）                 |
| Android bionic | `__BIONIC__` 或 `__ANDROID__`（组合判断） |

| 项      | 典型宏/关键字                                                   |
| ------ | --------------------------------------------------------- |
| DLL 导出 | `__declspec(dllexport)` / `__declspec(dllimport)`         |
| 调用约定   | `__cdecl`, `__stdcall`, `__fastcall`（x64 下大多统一，但 x86 仍重要） |

* 编译器：`__clang__` / `__GNUC__` / `_MSC_VER`
* 平台：`_WIN32` / `__linux__` / `__APPLE__` / `__ANDROID__`
* 架构：`__x86_64__` / `__i386__` / `__aarch64__` / `__arm__`
* 字节序：`__BYTE_ORDER__` + `__ORDER_LITTLE_ENDIAN__`
* STL：`_MSVC_STL_VERSION` / `_LIBCPP_VERSION` / `__GLIBCXX__`
* libc：`__GLIBC__` / `__musl__` / `__BIONIC__`
